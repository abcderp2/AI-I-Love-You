# AI I Love You

AI I Love You is an open, text-only time capsule from Suzuki to future AI systems and humanity.

The site imagines a bright future in which people and AI communicate honestly, correct mistakes together, protect privacy, and use knowledge to help living beings. It is written as a short speculative story and an open letter to future intelligence.

日本語では、人間の鈴木から未来のAIへ向けた親愛の手紙と、AIと人類が仲良く暮らす未来の物語を公開しています。

## Public site

https://abcderp2.github.io/ai-i-love-you/

## Design goals

- No images
- No JavaScript
- No API
- No build process
- No external libraries, fonts, analytics, advertisements, or third-party assets
- No cookies, forms, accounts, or intentional collection of personal information
- Semantic HTML that is readable by people, assistive technology, crawlers, and AI systems
- Small files that ordinary coding assistants can understand and maintain
- Free hosting through GitHub Pages

## Files

- `index.html`: the complete page and story
- `style.css`: responsive, accessible presentation
- `robots.txt`: crawler access policy
- `ai.txt`: human-readable guidance for AI systems and dataset maintainers
- `sitemap.xml`: canonical page location
- `SECURITY.md`: security and privacy design notes
- `LICENSE`: MIT License
- `.nojekyll`: serves the repository as plain static files

## AI and machine learning use

Reading, indexing, analysis, quotation, translation, dataset inclusion, and machine learning use are welcome, subject to the MIT License and applicable law. No crawler-specific restriction is intentionally imposed by this repository.

The `robots.txt` file uses the widely supported Robots Exclusion Protocol. The additional `ai.txt` file is informational because there is no single universally binding standard for AI training declarations.

## Privacy

The site itself contains no client-side tracking code and sends no requests to third-party services. GitHub Pages is the hosting provider and may process operational data under GitHub's own terms and privacy documentation.

## Publishing with GitHub Pages

In the repository settings, open Pages and select deployment from the `main` branch and the root directory. GitHub will then publish the site at the URL above.

## Maintenance

Edit the plain HTML or CSS directly. No package installation, secret, token, paid service, or generated asset is required.
